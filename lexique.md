**Architecture hexagonale** : Pattern de structuration avancé pour organiser le code de projets Go de manière robuste et maintenable.

**authentifications** : Processus de vérification de l'identité des utilisateurs dans une application web Go.

**autorisations** : Définition des droits d'accès et des permissions pour les utilisateurs authentifiés dans une application web Go.

**bases de données** : Systèmes pour stocker et récupérer des données.

**bases de données non relationnelles** : Type de bases de données, distinctes des bases relationnelles, pour la persistance des données.

**base relationnelle** : Type de base de données où les données sont structurées en tables, utilisée pour stocker et récupérer des données.

**bibliothèque SQL standard** : Méthode directe pour interagir avec une base de données relationnelle en exécutant des requêtes SQL.

**bonnes pratiques Web** : Ensemble de principes et techniques visant la sécurité, la structuration avancée et la robustesse des applications web.

**channels** : Mécanisme en Go pour la synchronisation et la communication sécurisée entre les goroutines.

**chat** : Exemple d'application Web temps réel où Go peut gérer la communication instantanée.

**clean architecture** : Pattern de structuration avancé pour organiser le code de projets Go de manière robuste et maintenable.

**communication entre goroutines** : L'échange d'informations et de données entre les goroutines, souvent facilité par les channels.

**Communication sécurisée entre goroutines** : L'échange d'informations fiable et sans conflit entre les goroutines, généralement via des channels.

**concurrence** : Capacité à exécuter plusieurs opérations simultanément ou en parallèle, gérée en Go par les goroutines et channels.

**Connexion à une base de données relationnelle** : Action d'établir une liaison entre une application Go et une base de données relationnelle, telle que PostgreSQL.

**CSRF (Cross-Site Request Forgery)** : Type d'attaque courante ciblant la sécurité des applications web.

**DELETE** : Type de requête SQL utilisée pour supprimer des données d'une base de données relationnelle.

**entités** : Représentations d'objets ou de structures de données manipulées par un ORM.

**Gestion des dépendances** : Processus de gestion des bibliothèques externes et des modules qu'un projet Go utilise.

**Gestion des erreurs** : Processus d'identification et de traitement des problèmes ou défaillances, par exemple lors de l'exécution de requêtes SQL.

**goroutines** : Unités d'exécution légères et simultanées en Go, utilisées pour gérer des opérations concurrentes.

**GORM** : Exemple d'ORM simple spécifiquement mentionné pour Go, facilitant la manipulation des entités.

**injection SQL** : Type d'attaque courante où du code SQL malveillant est inséré dans des entrées utilisateur pour manipuler une base de données.

**INSERT** : Type de requête SQL utilisée pour ajouter de nouvelles données à une base de données relationnelle.

**maintenabilité** : Qualité du code qui indique la facilité avec laquelle il peut être compris, modifié et corrigé au fil du temps.

**notifications** : Exemple d'application Web temps réel où Go peut gérer l'envoi d'alertes instantanées.

**opérations simultanées** : Plusieurs tâches ou processus qui sont exécutés en même temps, souvent gérés par les goroutines.

**ORM (Object-Relational Mapping)** : Technique ou bibliothèque permettant de manipuler des données de bases relationnelles en utilisant des objets de langage de programmation, sans écrire directement du SQL.

**Organisation du code** : La manière de structurer et d'arranger le code d'une application pour améliorer sa clarté et sa maintenabilité.

**patterns de structuration** : Modèles ou architectures d'organisation du code pour construire des projets robustes et maintenables.

**Persistance des données** : Processus de stockage et de récupération de données de manière durable, typiquement dans une base de données.

**requêtes SQL** : Instructions ou commandes utilisées pour interagir avec une base de données relationnelle (ex: SELECT, INSERT, UPDATE, DELETE).

**sanitation** : Processus de nettoyage des entrées utilisateur pour supprimer ou neutraliser tout contenu potentiellement dangereux.

**Sécurité** : Ensemble de mesures et de pratiques visant à protéger une application web Go contre les menaces et les vulnérabilités.

**Sécurisation des entrées utilisateur** : Mesures prises pour protéger une application contre les données malveillantes ou non valides fournies par les utilisateurs.

**SELECT** : Type de requête SQL utilisée pour récupérer des données d'une base de données relationnelle.

**structuration avancée** : Application de patterns d'organisation de code complexes pour rendre les projets Go plus robustes et maintenables.

**Synchronisation** : Mécanisme qui assure la bonne coordination et la cohérence entre différentes goroutines ou opérations concurrentes.

**tâches** : Opérations ou unités de travail qui peuvent être lancées et gérées par des goroutines.

**tests unitaires** : Méthode de test qui vérifie des petites parties (unités) du code de manière isolée pour s'assurer de leur bon fonctionnement.

**UPDATE** : Type de requête SQL utilisée pour modifier des données existantes dans une base de données relationnelle.

**validation** : Processus de vérification que les entrées utilisateur respectent les formats, types et contraintes attendus.

**Web temps réel** : Capacité d'une application web à traiter et à afficher des informations de manière instantanée, souvent via des mécanismes comme les WebSockets.

**Websocket** : Protocole de communication permettant une connexion persistante et bidirectionnelle, clé pour l'implémentation du Web temps réel en Go.

**XSS (Cross-Site Scripting)** : Type d'attaque courante où des scripts malveillants sont injectés dans des pages web visualisées par d'autres utilisateurs.
